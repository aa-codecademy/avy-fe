# User Stories — Students & Company

_Source: User stories Students.docx (original student-facing content was in Macedonian; translated for consistency.)_

## Student

### Register on the Avy platform

- As a student, I want to register on the Avy platform so that I can access it.

**Acceptance criteria**

- Successful registration on the platform
- On failed registration, return to the previous steps or deny authorization to use the platform

### Log in and browse jobs

- As a student, I want to log in to the Avy platform so that I can search for job openings.

**Acceptance criteria**

- Successful login to the platform
- On failed login, return to the login page

### Profile — qualifications and skills

- As a student, I want to create a profile so that my qualifications and skills are visible.

**Acceptance criteria**

- Profile created successfully according to predefined fields
- Ability to choose which job types I want notifications for
- Ability to log out of the platform
- Ability to change or add data
- Overview of all activity (applications and status)

### Password management

- As a student, I want to manage my password so that I can use the platform successfully.

**Acceptance criteria**

- Ability to change password while logged in
- Ability to reset a forgotten password when logging in

### Preferred language

- As a student, I want to select a preferred language so that I can fully understand the portal.

**Acceptance criteria**

- Ability to change preferred language with application-wide effect across all platform features

### Logo click → Home

- As a student, I want a clickable logo so that I can return to the home page quickly.

**Acceptance criteria**

- Clickable logo always available in the header, with navigation to the home page

### Job recommendations

- As a student, I want information about job openings that match my profile so that I can find suitable work or an internship faster.

**Acceptance criteria**

- Notification for a new job opening that matches what the student opted to be notified about
- Predefined algorithm that matches job requirements with the student’s stated qualifications

### Filter listings

- As a student, I want to filter listings by industry, skills, work mode, and engagement type (full-time / part-time / freelance / internship).

**Acceptance criteria**

- Search is visible on the home page
- Ability to select different search categories (job title, location, company, working hours, sector, education, experience, company rank, expiry date)
- Keyword search

### CV form visible to companies

- As a student, I want to complete a CV form (visible to companies) so that I can be contacted based on qualifications and education.

**Acceptance criteria**

- Predefined fields for the student profile (first and last name, age, education, work experience, expertise)
- Ability to change or add data

### Update profile

- As a student, I want to update my profile so that it stays up to date.

**Acceptance criteria**

- System allows changing or adding data; changes are visible immediately after saving

### Latest listings

- As a student, I want to see the newest listings so that I can browse faster.

**Acceptance criteria**

- Newly published listings show a “NEW” marker
- All new listings appear among the top / first on the home page

### “NEW” on a listing

- As a student, I want a “NEW” indicator on the listing so that I can identify new listings more easily when browsing.

**Acceptance criteria**

- Visible “NEW” marker on each newly published position, shown for 3 days from publication

### Updates from companies (applications)

- As a student, I want updates from companies where I have submitted an application.

**Acceptance criteria**

- Email notification for sent applications
- Email notification for status updates
- Additional area on the profile for all applications and their status, for current roles and full history

### Personalized notifications

- As a platform user, I want to choose whether I receive notifications for a personalized experience over time:
    - Messages
    - Interview invitation
    - Status for the job I applied to
    - Successful application
    - Successful password change
    - Successful registration

**Acceptance criteria**

- Notification by email or in-app

### Events and industry news

- As a platform user, I want information about upcoming employment-related events and current news so that I have a professional view of trends in my focus industries.

**Acceptance criteria**

- Notification by email or in-app

### Apply to a listing

- As a student, I want to apply to published listings so that I can complete a successful application.

**Acceptance criteria**

- Quick-apply option for each published listing
- Application using the predefined CV form or by uploading a CV document
- Ability to add extra documents or certificates
- In-page notification when the application succeeds
- Email and profile notification on successful application

### Application tracker

- As a student, I want to see each application with its status in the tracker so that I have a better overview of all applications.

**Acceptance criteria**

- Profile area with history of all current and past applications and a stated status per application (ongoing, rejected, accepted)

### Withdraw application

- As a student, I want to withdraw an application during recruitment so that I stay transparent.

**Acceptance criteria**

- Button to withdraw the application easily
- Notification to the company that a candidate withdrew
- Notification to the student that the application was withdrawn
- “Decline” status on the student’s profile for that application

### Message to company

- As a student, I want to send a message or comment to a company so that I can contact the company I want.

**Acceptance criteria**

- Field for message or comment addressed to the selected company
- Email notification for sent/received messages
- Pop-up icon for unread messages that opens platform messages

### Extra documents on application

- As a student, I want to add a CV or extra documents when applying.

**Acceptance criteria**

- On application, field to upload extra documents (CV form, cover letter, certificates)
- Under the defined field, show the uploaded document name and format
- Notification if the platform does not support that format
- Notification if the file exceeds the allowed size

### Easy Apply

- As a student, I want to apply via Easy Apply so that I can submit an application quickly.

**Acceptance criteria**

- Quick-apply option for each published listing
- Application using the predefined CV form or by uploading a CV document
- Ability to add extra documents or certificates
- In-page notification when the application succeeds
- Email and profile notification on successful application

### Comment when applying

- As a student, I want to leave a comment for the company when applying so that I can communicate and add context.

**Acceptance criteria**

- Field for message or comment addressed to the selected company

### Positions by selected company

- As a student, I want to select a company and see all open positions at that company so that I have a clearer view of openings.

**Acceptance criteria**

- Ability to filter by company in the search bar
- List of all open positions matching the selection in the search bar

## Company

- As a company, I want to register so that I can post job openings.
- As a company, I want to log in so that I can post job openings.
- As a company, I want to create a profile so that I am visible to potential applicants and alumni.
- As a company, I want to manage my password so that I can use the platform.
- As a company, I want to log in to a dedicated company area so that I enter the correct profile.
- As a company, I want to select a preferred language so that I can fully understand the portal.
- As a company, I want a clickable logo so that I can return to the home page quickly.
- As a company, I want a predefined form for company data and contact person so that students can understand the company clearly.
- As a company, I want to open, edit, or close job announcements so that I can recruit suitable staff.
- I want to write listings in a predefined form so that they are easy to understand.
- As a company, I want notifications for:
    - Successful registration
    - Successful password change
    - Successful candidate application for a published posting
    - Each incoming application

- As a company, I want to see all students who applied to a posting so that I can review candidates.
- As a company, I want to search students by skills so that I can recruit those who match the requirements.
- As a company, I want to message students to invite them to interviews or inform them of status.
- As a company, I want to choose whether applicants use Easy Apply or must attach a CV form, matching recruiter needs.
