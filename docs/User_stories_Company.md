# Company — User Stories

_Source: User stories Company.docx_

## EPIC 1: Company Account Management

### User Story 1.1 (Create Company Account)

- **As a** company
- **I want to** register an employer account
- **So that** I can access hiring tools

**Acceptance criteria**

- Company can sign up using email
- Required fields: company name, industry, size
- Email verification is completed
- Employer dashboard is created

### User Story 1.2 (Secure Access)

- **As a** company user
- **I want to** securely log into the platform
- **So that** I can manage hiring activities

**Acceptance criteria**

- Login with email/password
- Password reset available
- Secure session handling

## EPIC 2: Company Profile Management

### User Story 2.1 (Manage Company Profile)

- **As a** company
- **I want to** maintain my company profile
- **So that** I can present my brand to candidates

**Acceptance criteria**

- Add/edit company description, logo, website
- Display active job postings
- Save and update information

## EPIC 3: Job Posting & Control

### User Story 3.1 (Create Job Listing)

- **As a** company
- **I want to** post job openings
- **So that** I can attract candidates

**Acceptance criteria**

- Add job title, description, requirements, salary
- Select job type and location
- Publish job listing

### User Story 3.2 (Edit Job Listing)

- **As a** company
- **I want to** update job postings
- **So that** I keep information accurate

**Acceptance criteria**

- Edit all job fields
- Save changes instantly

### User Story 3.3 (Manage Job Status)

- **As a** company
- **I want to** control job visibility
- **So that** I can open or close positions

**Acceptance criteria**

- Mark job as active/inactive
- Close job when position is filled

## EPIC 4: Candidate Management

### User Story 4.1 (View Applicants)

- **As a** company
- **I want to** see all applicants for a job
- **So that** I can evaluate candidates

**Acceptance criteria**

- View list of applicants per job
- Access candidate profiles and resumes

### User Story 4.2 (Filter & Sort Candidates)

- **As a** company
- **I want to** filter candidates
- **So that** I can quickly find suitable applicants

**Acceptance criteria**

- Filter by skills, experience, or status
- Sort by relevance or application date

### User Story 4.3 (Candidate Evaluation)

- **As a** company
- **I want to** manage candidate status
- **So that** I can track hiring progress

**Acceptance criteria**

- Mark candidates as shortlisted, rejected, or hired
- Update status in real time

## EPIC 5: Hiring Workflow Management

### User Story 5.1 (Track Hiring Pipeline)

- **As a** company
- **I want to** track candidate progress
- **So that** I have visibility over hiring stages

**Acceptance criteria**

- Pipeline view (Applied → Shortlisted → Interview → Hired)
- Status updates reflected instantly

### User Story 5.2 (Internal Notes)

- **As a** company
- **I want to** add notes to candidates
- **So that** I can collaborate internally

**Acceptance criteria**

- Add/edit notes per candidate
- Notes visible to authorized team members only

## EPIC 6: Communication

### User Story 6.1 (Contact Candidates)

- **As a** company
- **I want to** communicate with candidates
- **So that** I can move them through the hiring process

**Acceptance criteria**

- Send messages to candidates
- Receive notifications on replies

### User Story 6.2 (Notifications)

- **As a** company
- **I want to** receive alerts
- **So that** I stay updated on applications

**Acceptance criteria**

- Notifications for new applications
- Alerts for candidate responses
